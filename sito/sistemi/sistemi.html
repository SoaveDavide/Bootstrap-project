Che cosa sono? A che cosa servono? Come funzionano? Esempio di utilizzo

Che cosa sono?

Una VLAN (Virtual Local Area Network) è una rete logica creata su una rete fisica per isolare e separare i dispositivi, migliorando sicurezza e gestione. Permette a dispositivi su switch diversi di comunicare come se fossero sulla stessa rete. Dispositivi in VLAN diverse non comunicano direttamente senza un router. Le VLAN riducono il traffico di rete, aumentano la sicurezza e semplificano la configurazione della rete aziendale.

A che cosa servono?

Una VLAN serve:

I dispositivi all'interno di una VLAN possono comunicare solo tra loro, mentre quelli in VLAN diverse restano isolati, a meno che non sia presente un router o un dispositivo Layer 3 configurato per interconnetterle. Questo migliora la sicurezza della rete.

Limita il traffico broadcast e multicast a una specifica VLAN, evitando che tutto il traffico venga propagato su tutta la rete fisica. Ciò migliora le prestazioni, specialmente su reti di grandi dimensioni.

Consente di raggruppare dispositivi logicamente in base alla funzione (es. amministrazione, vendite, IT), indipendentemente dalla loro posizione fisica, semplificando la gestione della rete.

Organizza meglio reti aziendali complesse, riducendo il bisogno di cablaggi separati per ogni gruppo o funzione e permettendo modifiche rapide.

Come funzionano?

Una VLAN (Virtual Local Area Network) funziona separando logicamente i dispositivi all'interno di una rete fisica in gruppi distinti. Ogni VLAN agisce come una rete indipendente, consentendo ai dispositivi al suo interno di comunicare tra loro, ma isolandoli dalle altre VLAN, salvo configurazioni specifiche tramite router o switch Layer 3. Gli switch gestiscono le VLAN assegnando le porte a specifici ID VLAN. Quando un dispositivo invia un pacchetto, lo switch lo contrassegna con un tag VLAN (tramite il protocollo IEEE 802.1Q) per identificarne la rete logica. Solo i dispositivi della stessa VLAN possono ricevere questi pacchetti, riducendo il traffico broadcast e migliorando la sicurezza. Router o dispositivi Layer 3 sono necessari per permettere la comunicazione tra VLAN diverse, configurando regole specifiche (routing inter-VLAN). Questo sistema è ideale per gestire reti aziendali complesse e ottimizzare risorse e prestazioni.

COMANDI PER CONFIGURARE VLAN


Switch> enable                // Entra in modalità privilegiata
Switch# configure terminal    // Entra in modalità di configurazione globale

Switch(config)# vlan 10       // Crea la VLAN 10
Switch(config-vlan)# name Sales // Assegna il nome "Sales" alla VLAN 10
Switch(config-vlan)# exit      // Torna alla modalità di configurazione globale

Switch(config)# interface range fa0/1 - 10 // Seleziona un intervallo di porte (ad esempio fa0/1 a fa0/10)
Switch(config-if-range)# switchport mode access  // Imposta la modalità delle porte su "access"
 Switch(config-if-range)# switchport access vlan 10  // Assegna la VLAN 10 alle porte selezionate
Switch(config-if-range)# exit  // Torna alla modalità di configurazione globale
 
Switch(config)# interface fa0/11 // Seleziona un'altra porta (ad esempio fa0/11)
Switch(config-if)# switchport mode trunk  // Imposta la porta in modalità trunk (per trasportare traffico di più VLAN)
Switch(config-if)# switchport trunk allowed vlan 10,20  // Consenti solo VLAN 10 e VLAN 20 sulla porta trunk
Switch(config-if)# exit  // Torna alla modalità di configurazione globale

Switch(config)# exit  // Esci dalla modalità di configurazione globale
Switch# write memory  // Salva la configurazione
                                    


Router# enable                    // Accedere alla modalità privilegiata
Router# configure terminal         // Entrare in modalità di configurazione globale
Router(config)# interface fa0/0    // Selezionare l'interfaccia di rete (ad esempio fa0/0)
Router(config-if)# ip address 192.168.1.1 255.255.255.0  // Impostare l'indirizzo IP e la subnet mask
Router(config-if)# no shutdown     // Abilitare l'interfaccia
Router(config-if)# exit            // Uscire dalla configurazione dell'interfaccia
Router(config)# exit               // Uscire dalla modalità di configurazione globale   
                                


Switch# enable                    // Entrare in modalità privilegiata
Switch# configure terminal         // Entrare in modalità di configurazione globale
Switch(config)# vlan 10            // Creare la VLAN 10
Switch(config-vlan)# name Sales    // Dare un nome alla VLAN
Switch(config-vlan)# exit          // Uscire dalla configurazione della VLAN
Switch(config)# interface vlan 10  // Selezionare l'interfaccia VLAN 10
Switch(config-if)# ip address 192.168.10.1 255.255.255.0 // Impostare l'indirizzo IP della VLAN 10
Switch(config-if)# no shutdown     // Abilitare l'interfaccia VLAN
Switch(config-if)# exit            // Uscire dalla configurazione dell'interfaccia
Switch(config)# exit               // Uscire dalla configurazione globale
